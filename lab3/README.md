## Формулировка задания

Составить программу выполнения заданных преобразований арифметических выражений с применением деревьев. Преобразование выражения и дерево реализуются осуществлять одним из известных методов (Узгера, Дейкстры и др.). Операнда в обрабатываемом выражении могут быть целого или вещественного типа. Заданы унарные и бинарные операции над операндами и для буквенно-цифровых переменных. Преобразование выражения реализовать в виде набора подпрограмм. Программа должна воспринимать исходное выражение в инфиксной записи, преобразовывать его в дерево, выполнять заданные преобразования внутри дерева и выводить результат в виде дерева и в текстовом представлении.

Для некоторых задач приводятся примеры, поясняющие постановку задачи. В примерах слева от стрелки приводится фрагмент выражения до, а справа — после выполнения преобразования. Программа должна находить все вхождения подобных фрагментов и выполнять соответствующие преобразования, а не только те, что приведены в примере.

Программу необходимо проверить на нескольких выражениях, среди которых должны быть выражения, не содержащие преобразуемых элементов, содержащие ровно один такой элемент или несколько элементов, подлежащих преобразованию, причем в «гнёздах» между деревья.

**Вариант 26:** Заменить степень с разностью в показателе на частное степеней:

```
a ^ (b - c) → a ^ b / a ^ c
```

## Описание выполненной работы

### Структура проекта

Проект состоит из следующих файлов:

#### 1. `expression_tree.c` - Основная программа и реализация алгоритмов

Содержит структуры `Node`, `Token` и все функции для работы с деревом выражений, реализацию парсинга, преобразования и вывода.

**Структуры данных:**

```c
typedef struct Node {
    char value[MAX_TOKEN_LEN];
    struct Node *left;
    struct Node *right;
} Node;

typedef struct {
    TokenType type;
    char value[MAX_TOKEN_LEN];
} Token;
```

**Основные функции:**

- `create_node(value)` - создание нового узла
- `copy_node(node)` - глубокое копирование узла
- `free_tree(node)` - освобождение памяти дерева
- `tokenize(expression, tokens)` - разбиение строки на токены
- `parse_expression(expression)` - парсинг инфиксного выражения в дерево с использованием алгоритма сортировочной станции
- `transform(node)` - преобразование выражения согласно варианту 26
- `to_infix(node)` - получение инфиксной записи из дерева
- `print_tree(node)` - визуальное представление дерева в виде иерархии
- `get_precedence(op)` - получение приоритета оператора
- `is_left_associative(op)` - проверка ассоциативности оператора
- `is_leaf(node)` - проверка, является ли узел листом
- `apply_operator()` - применение оператора к операндам

**Алгоритм преобразования:**
1. Рекурсивно обходим дерево снизу вверх
2. Для каждого узла проверяем соответствие паттерну:
   - Узел является оператором `^`
   - Правый дочерний узел является оператором `-`
   - Левый дочерний узел существует
3. Если паттерн найден:
   - Создаем копию левого поддерева (основание степени `a`)
   - Создаем два новых узла возведения в степень: `a ^ b` и `a ^ c`
   - Создаем узел деления, объединяющий эти два выражения
4. Возвращаем новое поддерево

**Поддерживаемые операторы:**
- `+` - сложение (приоритет 1, левоассоциативный)
- `-` - вычитание (приоритет 1, левоассоциативный)
- `*` - умножение (приоритет 2, левоассоциативный)
- `/` - деление (приоритет 2, левоассоциативный)
- `^` - возведение в степень (приоритет 3, правоассоциативный)

## Примеры работы

### Пример 1
**Вход:** `a ^ (b - c)`  
**Выход:** `a ^ b / a ^ c`

### Пример 2
**Вход:** `2 ^ (3 - 1)`  
**Выход:** `2 ^ 3 / 2 ^ 1`

### Пример 3
**Вход:** `x ^ (y - z) + 5`  
**Выход:** `x ^ y / x ^ z + 5`

### Пример 4
**Вход:** `(a + b) ^ (c - d)`  
**Выход:** `(a + b) ^ c / (a + b) ^ d`

### Пример 5 (без преобразования)
**Вход:** `a ^ b - c`  
**Выход:** `a ^ b - c` (не соответствует паттерну)

### Пример 6 (без преобразования)
**Вход:** `a ^ (b + c)`  
**Выход:** `a ^ (b + c)` (не разность в показателе)
