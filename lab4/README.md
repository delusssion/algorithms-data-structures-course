## Формулировка задания

Построить упорядоченное дерево для хранения и поиска элементов с уникальными строковыми ключами и вещественными значениями. Ключ — непустая последовательность латинских букв не более чем из 6 символов. Ключи упорядочиваются лексикографически.

Вид дерева определяется формулой `(N % 3) + 1`, где N = 24:
- (24 % 3) + 1 = **1 — AVL-дерево**

Для дерева реализовать 4 операции:
1. Добавление узла по ключу (с балансировкой)
2. Удаление узла по ключу (с балансировкой)
3. Печать дерева
4. Поиск значения по ключу

Предусмотреть ввод команд из текстового файла и вывод результатов в текстовый файл.

## Описание выполненной работы

### Структура проекта

Проект состоит из следующих файлов:

#### 1. `avl_tree.c` — основная программа и реализация AVL-дерева

**Структура данных:**

```c
typedef struct AVLNode {
    char key[MAX_KEY_LEN];   // ключ: 1-6 латинских букв
    double value;            // вещественное значение
    struct AVLNode *left;
    struct AVLNode *right;
    int height;              // высота поддерева для балансировки
} AVLNode;
```

**Основные функции:**

- `avl_create_node(key, value)` — создание нового узла
- `avl_insert(node, key, value)` — вставка с балансировкой; если ключ уже существует — обновляет значение
- `avl_delete(node, key, found)` — удаление с балансировкой
- `avl_search(node, key)` — поиск узла по ключу
- `avl_print(root, out)` — визуальная печать дерева в файл/stdout
- `avl_free(node)` — освобождение памяти
- `avl_height(node)` — высота узла (0 для NULL)
- `avl_balance_factor(node)` — фактор баланса (высота левого − правого)
- `is_valid_key(key)` — проверка корректности ключа (1-6 латинских букв)

**Алгоритм балансировки AVL-дерева:**

После каждой вставки и удаления пересчитывается высота и проверяется фактор баланса `bf = h(left) - h(right)`.

| Ситуация | Условие | Действие |
|---|---|---|
| Левый перекос, левое поддерево | bf > 1, bf(left) >= 0 | Правый поворот |
| Левый перекос, правое поддерево | bf > 1, bf(left) < 0 | Левый поворот левого, затем правый |
| Правый перекос, правое поддерево | bf < -1, bf(right) <= 0 | Левый поворот |
| Правый перекос, левое поддерево | bf < -1, bf(right) > 0 | Правый поворот правого, затем левый |

**Алгоритм удаления:**

Если удаляемый узел имеет двух потомков, он заменяется симметричным преемником (минимальным элементом правого поддерева), после чего этот преемник удаляется из правого поддерева с последующей балансировкой.

**Формат входного файла:**

```
1 key value    — добавить/обновить узел
2 key          — удалить узел
3              — напечатать дерево
4 key          — найти значение по ключу
```

**Формат выходного файла:**

```
<команда>
<результат>

<команда>
<результат>
```

**Запуск:**

```
./main input.txt output.txt    — читать из файла, писать лог в файл
./main input.txt               — читать из файла, лог не писать
./main                         — интерактивный режим
```

### Пример 2 — дерево из 24 узлов (`input.txt`)

После добавления 24 греческих букв дерево автоматически балансируется. AVL-балансировка гарантирует, что высота дерева остаётся O(log n) — при 24 узлах высота не превышает 5.
